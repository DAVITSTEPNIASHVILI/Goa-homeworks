უპასუხეთ შემდეგ კითხვებს:
  1.რა არის django, რისთვის შეიძლება გამოვიყენოთ
  django არის framework რომელიც უზრუნველყოფს front-end და back-end საც.

  2.რა განსხვავებაა როცა django ს ვიყენებთ როგორც full stack framework ს და როგორც back end framework ს
  როდესაც django-ს ვიყენებთ full stack-ისთვის მაშინ შეგვიძლია მხოლოდ ორ მეთოდზე მუშაობა: GET-ზე და POST-ზე, ხოლო back end-ისთვის ყველა მეთოდზე.

  3.რა უპირატესობები აქვს django ს სხვა stackებთან შედარებით და რა ნაკლი აქვს ( იგივე MERN stack თან შედარებით, რესურსებში ჩაგდებულ ვებსაიტზე django-tutorial-dev ში შეგიძლიათ დეტალურად ნახოთ შედარება სხვადასხვა სხვა stack ებთნა )
  django-ს უკვე ბევრი ფუნქციონალი აქვს ჩაშენებული, მაგალითად: ინფორმაციის mail-ზე გაგზავნა და ა.შ. (ნელია)

  4.რა არქიტექტურას მოიხმარს django
  django მოიხმარს mvt იგივე model, view, template სტრუქტურას, რომელიც ძალიან მარტივია.

  5.რა არის model view template
  model - აქ ვქმნით მონაცემთა ბაზის სტრუქტურას.
  view - აქ ვმუშაობთ request-ებზე და response-ებზე
  template - ეს არის იგივე front-end

  6.რა არის ORM და რატომაა კარგი მისი გამოყენება
  ORM ნიშნავს object relational mapping. მონაცეემთა ბაზის კოდის წერა კონკრეტულად როცა ვმუშაობთ sql თან ჩვენ არ გვიწევს.

  7.როდის უნდა გამოვიყენოთ django და როდის უფრო light stack ი
  django-ს ვიყენებთ უფრო მძიმე პროექტებზე, ხოლო light stack-ს ვიყენებთ უფრო მოკლე პროექტებზე.

  8.რა არის აპლიკაცია django ში და რის მიხედვით ვყოფთ აპლიკაციებს
  აპლიკაციებს ჩვენ ვიყენებთ იმისთვის რომ ვუზრუნველყოთ სხვადასხვა ოპერაციები. თითო აპლიკაცია უნდა უზრუნველყოფდეს კონკრეტული ერთი ინფორმაციის ნაწილაკის კონტროლს.

  9.რა არის pipenv და რატომაა უკეთესი მისი გამოყენება
  pipenv-ის საშუალებით შეგვიძლია დავაყენოთ django.
